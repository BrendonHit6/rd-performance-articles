## Slide 5 — Node.js 24: Native TypeScript

- Node 24 (поточний LTS) — `node app.ts` просто працює, жодних залежностей.
- Коротко озвучити таймлайн: Node 22 (прапорець) → Node 23 (без прапорця) → Node 24 LTS (за замовчуванням) → Node 25.2 (стабільно).
- Під капотом: модуль `amaro` (обгортка SWC) — це **стрипер типів**, не компілятор.
- Ключова відмінність від `tsx`/`ts-node`: живе всередині рантайму, а не як окрема залежність.

---

## Slide 6 — How Type Stripping Works

- **Типи трактуються як коментарі**: Node замінює анотації на пробіли — номери рядків не зміщуються, source maps не потрібні, стектрейси коректні.
- **Критичне застереження — повторити двічі**: перевірки типів під час виконання немає. `add("hello", "world")` за сигнатури `add(a: number, b: number)` виконається і поверне `"helloworld"`.
- Стрипінг розділяє те, що `tsc` завжди об'єднував: **валідація типів** → `tsc --noEmit` у CI; **виконання** → Node нативно.

---

## Slide 7 — Type Stripping: Limitations You Must Know

- **enum / namespace** генерують рантайм-код → потрібен `--experimental-transform-types`. Рекомендація: мігрувати на `as const`.
- **tsconfig.json ігнорується** — `paths`, downleveling, декоратори не працюють.
- **`import type` обов'язковий** — тип як value-імпорт не видаляється → падіння в рантаймі. Увімкніть `verbatimModuleSyntax: true`.
- **Немає JSX** — фіча для бекенду, скриптів, CLI. Фронтенд — через Vite.
- **`erasableSyntaxOnly: true`** (TS 5.8+) — забороняє несумісний синтаксис на етапі компіляції.

## Slide 8 — Why We Care: Cold Start

- Express-сервер ~600 рядків: `ts-node` ≈1,4 с → нативний стрипінг <200 мс — **розрив ~9×**.
- Для serverless кожен cold start тарифікується й впливає на UX — різниця напряму вимірювана у вартості.
- Додатково: менше dev-залежностей, простіші Dockerfile, швидший локальний цикл.

## Slide 9 — A Practical Setup for Our Projects

- `node --watch src/index.ts` — dev і скрипти: запускати `.ts` напряму, без компіляції.
- `tsc --noEmit` — обов'язкові ворота в CI, не замінюються нативним стрипінгом.
- `tsc` — залишається для npm-пакетів і продакшн-артефактів.
- Express і Fastify — без змін, працюють з нативним стрипінгом одразу.

---

## Slide 10 — TypeScript 7 "Corsa": Rewritten in Go

- Microsoft портує компілятор + мовний сервіс на **Go**. RC вийшов 18 червня 2026, стабільний реліз — у найближчі тижні.
- Числа: ~**10×** швидша компіляція, ~**8×** швидший старт LSP, **−50%** пам'яті.
- **TS 6.x** — перехідна версія на старій кодовій базі: увімкніть `--deprecation`, сплануйте міграцію від `baseUrl`, `moduleResolution: node`, ES5-таргетів.
- Вже зараз: протестувати `@typescript/native-preview` у неблокуючій CI-джобі.
- Головна теза: TypeScript більше не «мова, що компілюється в JS» — він стає діалектом, який рантайми розуміють нативно.

## Slide 11 — Section 02: ES2026

- Перехідний слайд. Одна фраза: «TC39 схвалив ES2026 у березні 2026 — рік великих фіч.»

---

## Slide 12 — Temporal: Date Finally Retires

- **Stage 4, ES2026** — заміна 30-річному `Date`: незмінні типи, вбудовані часові пояси й календарі.
- `PlainDate`, `PlainTime`, `ZonedDateTime`, `Instant`, `Duration` — кожна операція повертає новий об'єкт, без мутацій.
- Наслідок для команди: прибрати date-бібліотеки (moment / date-fns / luxon) у новому коді; виграш у розмірі бандла.
- Для старіших таргетів — `@js-temporal/polyfill`.

---

## Slide 13 — Quality-of-Life Wins: isError & fromAsync

- **`Error.isError(v)`** — `instanceof Error` не спрацьовує між realms (iframes, workers). Це `Array.isArray`, але для помилок.
- **`Array.fromAsync(src)`** — збирає асинхронний ітерабл у масив одним викликом. Те саме, що ми пишемо вручну через `for await...of` у кожному сервісі сьогодні.

---

## Slide 14 — import defer: Lazy Module Initialization

- **Важливо:** `import defer` досяг Stage 4 у березні 2026, але **пропустив snapshot ES2026** — офіційно це **ES2027**. Браузерні реалізації вже є (Firefox, Chrome у процесі).
- Модуль завантажується, але не виконується до першого звернення до namespace.
- На відміну від `import()`: доступ залишається **синхронним** — без промісів, без рефакторингу.
- Актуально для великих кодових баз із глибокими графами модулів (сотні мс на старті).

---

## Slide 15 — Map.getOrInsert & using

- **`Map.getOrInsert(key, default)`** — Stage 4, **ES2026**. Замінює конструкцію has/set/get одним пошуком. Класика: кеші мемоізації, групування.
- **`using` / `await using`** — автоматичний виклик `[Symbol.dispose]()` при виході зі скоупу, включно з викиданням помилок. Це try/finally, який неможливо забути написати. Вже доступно в Chrome, Node, Deno, TypeScript, Babel — **частина ES2025**, не нова фіча.

## Slide 17 — Vite — The De Facto Standard

- **130M+** щотижневих завантажень; 98% задоволеності розробників (Webpack: 26%).
- React, Vue, Svelte, Shopify — усі обрали Vite дефолтом. Міграція з Webpack: 1–2 дні.
- **Vite 8 + Rolldown**: Rust-бандлер замінює розділення esbuild(dev)/Rollup(prod) — один рушій.
- **Cloudflare придбала VoidZero** (4 червня 2026) — довгострокове фінансування та корпоративна підтримка забезпечені.

---

## Slide 18 — Bun: The Runtime Anthropic Bought

- **Anthropic придбала Bun** у грудні 2025. У травні 2026 AI переписав ~960 тис. рядків коду з Zig на Rust за 6 днів (13 044 unsafe-блоки — інженерна реальність складніша, ніж заголовок).
- 1,52M щотижневих завантажень, ~92K GitHub stars; **16%** JS-розробників запускали Bun у продакшні.
- Наше позиціювання: **`bun install` і `bun test` — drop-in виграші, пробуйте зараз**. Як основний рантайм — спочатку пілот на некритичному сервісі.

---

## Slide 19 — Biome: Replacing ESLint + Prettier

- Лінтер + форматер в одному Rust-бінарнику; **~56×** швидший за ESLint; typed rules без запуску TS-компілятора (Biome v2).
- Міграція автоматизована: `biome migrate eslint` / `biome migrate prettier`.
- Аргумент поза швидкістю: заміна ESLint + Prettier + десяток плагінів на одну залежність зменшує поверхню supply-chain атак.
- Застереження: покриття правил не 1:1 з екосистемою плагінів ESLint — перевірте список перед міграцією.

## Slide 20 — Section 04: Frameworks

- Перехідний слайд. Одна фраза: «Server-first, дрібнозерниста реактивність і кінець важкої гідратації.»

## Slide 21 — Frameworks Are Converging on Four Ideas

- **Сигнали** замість virtual DOM diffing: оновлюється тільки те, що змінилось.
- **Server-first**: RSC, SSR streaming, edge — менше JS у браузері.
- **Компіляторна оптимізація**: React Compiler, Vue Vapor, Svelte 5 — робота переноситься на build-time.
- **AI-assisted workflows**: фреймворки спрощують абстракції, щоб code generators видавали надійний результат.
- Вибір фреймворка у 2026 — про **відповідність команді й компроміси**, а не сиру спроможність.

---

## Slide 22 — React 19: Compiler + Server Components

- **React Compiler**: авто-мемоізує, скорочує зайві ре-рендери на ~25–40%, прибирає більшість ручних `useMemo`/`useCallback`.
- **Server Components**: первинний рендер 2,4 с → 0,8 с (~67% швидше); ~45% нових проєктів стартують на RSC.
- Чесно про ризики: Context API не перетинає серверну межу; дебаг server/client-межі складний; Next.js lock-in (альтернативи: TanStack Start, React Router v7).

---

## Slide 23 — Signals Become the Standard (Beyond React)

- **Angular 21**: сигнали + zoneless — бандли ~18% легші без zone.js. Міграція у великих кодових базах — багатоспринтова.
- **Vue 3.6 Vapor Mode**: компіляція без virtual DOM — вражаюча швидкість монтування.
- **Svelte 5 Runes** (`$state`, `$derived`): явна реактивність; є офіційний codemod.
- **SolidJS**: еталонна реалізація дрібнозернистості; в парі з Astro islands — мінімум клієнтського JS.
- **TC39 Signals** — реактивність може увійти в мову: стежити, але поки не будувати на цьому.

---

## Slide 24 — The End of Heavy Hydration

- **Класична гідратація**: весь JS завантажується заздалегідь; сторінка «мертва» до пробудження — роздутий бандл → поганий INP.
- **Islands / fine-grained (Astro)**: JS тільки у інтерактивних островів; ідеально для контентних сайтів.
- **Resumability (Qwik)**: нуль JS на старті; код завантажується в момент взаємодії.
- Прив'язка до бізнесу: менше JS → кращий INP (Core Web Vital) → кращий SEO і утримання — це аргумент, який затверджує бюджет.

## Slide 26 — AI in the Workflow — No Longer Optional

- **46%** нового коду згенеровано AI; Copilot, Claude Code, v0 — стандарт для скафолдингу, тестів, рутинних задач.
- Балансуюча нотатка: спільнота вже обговорює, чи не пройшов «vibe coding» пік; якість обмежена ясністю кодової бази й розумінням рев'ювера.
- **Базові правила команди:**
  1. TypeScript обов'язковий — типи тримають і людей, і AI чесними.
  2. AI прискорює написання, але не замінює архітектуру і рев'ю — автор PR відповідає за код.
  3. Згенерований код — ті самі ворота: `tsc --noEmit`, Biome, тести, людське рев'ю.

## Slide 27 — Security: The Year npm Got Scary

- **CVE-2025-55182 (React2Shell)**: RCE у Next.js, CVSS **10.0** — вразливість десеріалізації RSC Flight; стандартний `create-next-app` вразливий без змін коду.
- **npm supply chain**: хвилі автоматизованих компрометацій 2025–2026 (Shai-Hulud), шкідливі postinstall-скрипти, тайпсквоти.
- **Контртренд zero-dependency**: ES2026 API (Temporal, fromAsync, using, Uint8Array codecs) нативно замінюють lodash/moment → менший бандл і менша поверхня атаки.
- **Наш мінімум (зробіть зобов'язанням):** lockfile audit у CI, запінені версії, негайне патчування фреймворків, мінімальні install-скрипти.

## Очікувані питання (Q&A prep)

**«Чи можемо ми повністю видалити build-крок?»**
Для скриптів, CLI і внутрішніх сервісів — так, вже сьогодні. Для npm-пакетів — ні, постачайте `.js`. Для продакшну, критичного до затримок, — заміряйте.

**«Якщо Node запускає TS, навіщо тримати `tsc`?»**
Node видаляє типи, але не перевіряє їх. `tsc --noEmit` — ворота коректності. З Corsa ця перевірка стане майже миттєвою, пропускати її буде ще менше причин.

**«Чи варто переписувати сервіси на Bun?»**
Ніякого переписування. Спочатку пакетний менеджер і тест-раннер (drop-in), рантайм — тільки через пілот.

**«Чи варті Server Components болю міграції?»**
Для наявних SPA — зазвичай ні. Для нових проєктів з SEO/INP-тиском — так. React Compiler, навпаки, вигідний усім майже без витрат на міграцію.

**«Temporal реально доступний?»**
Stage 4, ES2026; постачається в Chrome 144+, Firefox 139+. Поліфіл для відстаючих — `@js-temporal/polyfill`. Вже безпечно стандартизувати для нового коду.

**«46% AI-коду — нам потрібно менше розробників?»**
Зусилля зміщуються зі написання на специфікування, рев'ю й архітектуру. Наші ворота (типи, лінт, тести) — саме те, що тримає ті 46% активом, а не пасивом.

**«Яка одна зміна з найвищим ROI?»**
Biome + нативний TS dev-цикл. Обидва майже безризикові, займають по дню, прискорюють кожного розробника щодня.
